  Demo OpenShift TAM Interview | Red Hat

Mục đích project
----------------
Project demo 3-tier (Frontend → API → PostgreSQL) mô phỏng dashboard giao dịch
ngân hàng. Dùng để trình bày trong 20-30 phút:
  1. Deploy ứng dụng lên OpenShift
  2. Scale tự động (HPA)
  3. Giám sát qua OpenShift Console
  4. CI/CD deploy tự động

Chiến lược phát triển
---------------------
  Bước 1: Test LOCAL bằng Docker Compose (docker-compose.yml)
  Bước 2: Deploy lên OpenShift Local bằng manifest trong openshift/
  Bước 3: Setup CI/CD qua GitHub Actions (.github/workflows/)

Câu trả lời ngắn: CÓ — làm Docker trước rồi deploy OpenShift sau là cách đúng.
Docker Compose giúp debug nhanh; manifest OpenShift dùng cùng Dockerfile.

PHẦN 1 — KIẾN TRÚC TỔNG QUAN
================================================================================

  [Browser]
      |
      v
  [Frontend Pod - nginx:8080]  ----proxy /api---->  [Backend Pod - FastAPI:8000]
                                                              |
                                                              v
                                                    [PostgreSQL Pod - :5432]

Luồng request (FLOW)
--------------------
  1. User mở browser → truy cập Frontend (port 8080 local / Route trên OCP)
  2. Frontend serve file tĩnh: index.html, app.js, style.css
  3. app.js gọi API qua nginx reverse proxy:
       GET  /api/transactions      → danh sách giao dịch
       GET  /api/stats             → thống kê dashboard
       POST /api/transactions/generate → tạo giao dịch ngẫu nhiên (demo scale)
       GET  /health                → health check (OpenShift probe)
       GET  /ready                 → readiness (check DB)
       GET  /metrics               → metrics cơ bản
  4. Backend đọc/ghi PostgreSQL qua SQLAlchemy
  5. Khi load cao → HPA tăng số pod backend → demo scale trên Console

Service Discovery (OpenShift/K8s)
---------------------------------
  Trong cluster, các service gọi nhau qua DNS nội bộ:
    banking-api:8000     → Backend Service
    postgresql:5432      → Database Service
    banking-frontend:8080 → Frontend Service

  nginx.conf proxy_pass trỏ tới http://banking-api:8000 — tên trùng tên Service.

PHẦN 3 — BACKEND: CHI TIẾT CODE & FUNCTIONS
================================================================================

File: backend/app/database.py
-----------------------------
  DATABASE_URL  — Lấy từ env var, default local postgres
  engine        — SQLAlchemy engine kết nối PostgreSQL
  SessionLocal  — Factory tạo DB session
  Base          — Base class cho models
  get_db()      — Dependency injection cho FastAPI, tự đóng session

File: backend/app/models.py
---------------------------
  Class Transaction (bảng "transactions"):
    id            — Primary key, auto increment
    type          — "credit" | "debit" | "transfer"
    amount        — Số tiền (float)
    description   — Mô tả giao dịch
    account_from  — Tài khoản nguồn
    account_to    — Tài khoản đích
    status        — "completed" | "pending"
    created_at    — Timestamp

File: backend/app/schemas.py
----------------------------
  TransactionCreate   — Validate body POST /api/transactions
  TransactionResponse — Format JSON trả về client
  StatsResponse       — Format thống kê dashboard

File: backend/app/seed.py
-------------------------
  seed_transactions(db, count=20)
    — Chèn 20 giao dịch fake khi DB trống (chạy lúc startup)
  generate_random_transaction(db)
    — Tạo 1 giao dịch ngẫu nhiên, dùng cho nút "Generate Transaction"

File: backend/app/main.py — ENDPOINTS
-------------------------------------

  GET /health
    Công dụng: Liveness probe — OpenShift check pod còn sống không
    Response: { status, version, hostname }
    hostname giúp demo: khi scale, mỗi pod có hostname khác nhau

  GET /ready
    Công dụng: Readiness probe — pod chỉ nhận traffic khi DB OK
    Thực hiện: SELECT 1 trên PostgreSQL
    Response: { status: "ready", database: "connected" }

  GET /metrics
    Công dụng: Metrics cơ bản cho demo monitoring
    Response: { app_transactions_total, app_version_info, app_theme }

  GET /api/config
    Công dụng: Frontend đọc version + theme (đổi theme = rolling update demo)
    Env vars: APP_VERSION, APP_THEME (blue | red | green)

  GET /api/transactions?limit=20&offset=0
    Công dụng: Lấy danh sách giao dịch, mới nhất trước
    Response: Array of TransactionResponse

  POST /api/transactions
    Công dụng: Tạo giao dịch với body JSON hoặc không body (random)
    Body (optional): { type, amount, description, account_from, account_to }

  POST /api/transactions/generate
    Công dụng: Tạo nhanh 1 giao dịch random — nút trên UI + load test

  GET /api/stats
    Công dụng: 4 stat cards trên dashboard
    Response: { total_transactions, total_credit, total_debit, net_balance }

  lifespan (startup hook)
    — create_all tables nếu chưa có
    — seed 20 transactions ban đầu

Environment variables (Backend)
  DATABASE_URL  — postgresql://user:pass@host:5432/dbname
  APP_VERSION   — Hiển thị trên UI (default "1.0.0")
  APP_THEME     — "blue" | "red" | "green" — đổi màu header (demo rolling update)

PHẦN 6 — OPENSHIFT DEPLOY
================================================================================

Thứ tự apply manifest
-----------------------
  oc login ... (xem scripts/crc-setup.ps1)
  oc apply -f openshift/00-namespace.yaml
  oc apply -f openshift/01-postgresql.yaml
  oc wait --for=condition=ready pod -l app=postgresql -n banking-demo --timeout=120s

  # Build images (chọn 1 cách):

  Cách A — Binary build (không cần GitHub):
    oc new-build --name=banking-api --strategy=docker --binary=true -n banking-demo
    oc start-build banking-api --from-dir=./backend --follow -n banking-demo
    oc new-build --name=banking-frontend --strategy=docker --binary=true -n banking-demo
    oc start-build banking-frontend --from-dir=./frontend --follow -n banking-demo

  Cách B — Git build (sau khi push GitHub, sửa URL trong 05-buildconfigs.yaml):
    oc apply -f openshift/05-buildconfigs.yaml
    oc start-build banking-api -n banking-demo
    oc start-build banking-frontend -n banking-demo

  oc apply -f openshift/02-backend-deployment.yaml
  oc apply -f openshift/03-backend-hpa.yaml
  oc apply -f openshift/04-frontend-deployment.yaml

  # Lấy URL app:
  oc get route banking-frontend -n banking-demo

Chi tiết từng manifest OpenShift
---------------------------------

  00-namespace.yaml
    Tạo Namespace banking-demo — cô lập resources

  01-postgresql.yaml
    Secret postgresql-secret     — credentials + DATABASE_URL
    PVC postgresql-pvc           — disk 1Gi cho data
    Deployment postgresql        — 1 replica postgres:16-alpine
    Service postgresql           — ClusterIP port 5432

  02-backend-deployment.yaml
    Deployment banking-api
      - image: banking-api:latest (từ ImageStream sau build)
      - env DATABASE_URL từ Secret
      - readinessProbe: GET /ready
      - livenessProbe:  GET /health
      - resources requests/limits (CPU/RAM cho HPA + monitoring)
      - rollingUpdate maxUnavailable: 0 → zero downtime
    Service banking-api — ClusterIP port 8000

  03-backend-hpa.yaml
    HorizontalPodAutoscaler banking-api-hpa
      minReplicas: 1, maxReplicas: 5
      target CPU 50%
    Demo: chạy scripts/load-test.sh → xem pods tăng trên Console

  04-frontend-deployment.yaml
    Deployment banking-frontend — nginx + static
    Service banking-frontend — port 8080
    Route banking-frontend — URL public https://...apps-crc.testing
      TLS edge termination (OpenShift Route feature)

  05-buildconfigs.yaml
    ImageStream + BuildConfig cho backend và frontend
    Build từ Git repo (Docker strategy, dùng Dockerfile sẵn có)

PHẦN 9 — ĐỊNH NGHĨA THUẬT NGỮ (GLOSSARY)
================================================================================

  Pod          — Container nhỏ nhất chạy trên K8s/OpenShift
  Deployment   — Quản lý số replica + rolling update của pods
  Service      — DNS nội bộ, load balance tới pods
  Route        — OpenShift expose app ra internet (HTTPS)
  Namespace    — Project, không gian cô lập resources
  Secret       — Lưu password, connection string
  PVC          — PersistentVolumeClaim, disk gắn vào pod
  Probe        — Health check: liveness (restart) / readiness (traffic)
  HPA          — Horizontal Pod Autoscaler, scale theo CPU/RAM
  ImageStream  — Registry nội bộ OpenShift lưu container images
  BuildConfig  — Cấu hình build image từ source/Git
  Rolling Update — Cập nhật từng pod mới, không downtime
  S2I          — Source-to-Image (OpenShift build từ code không Dockerfile)
  OpenShift Local — Single-node cluster trên laptop (crc CLI)

PHẦN 10 — TROUBLESHOOTING
================================================================================

Docker local:
  - Port 5432/8000/8080 bị chiếm → đổi port trong docker-compose.yml
  - API crash → docker compose logs banking-api
  - DB chưa ready → đợi postgres healthcheck xanh

OpenShift Local:
  - crc start fail → chạy crc daemon ở terminal riêng
  - Không dùng PowerShell Admin cho crc commands
  - Cần cài trên ổ C:, Hyper-V enabled
  - Thiếu RAM → crc config set memory 14336 && crc stop && crc start

Deploy:
  - ImagePullBackOff → chưa build image, chạy oc start-build trước
  - CrashLoopBackOff API → check DATABASE_URL, postgres chưa ready
  - HPA không scale → cần metrics-server (có sẵn trên OpenShift)
  - Route không mở → oc get route -n banking-demo

Rolling update demo:
  - Đổi APP_THEME trong deployment env → apply → refresh browser
